# 実装手順書 Phase 3 静的サイト生成

> 注記
> 現行実装ではカテゴリページは `output/mock-site/categories/*.html`、商品詳細は `output/mock-site/products/*.html` の単一 HTML 出力で生成している。

## 1. 目的

取得済みテンプレート、商品データ、画像を使って、閲覧可能な静的モックサイトを生成する。

## 2. 開始条件

- Phase 2 が完了していること
- 1 件以上の商品の JSON と画像が取得できていること

## 3. 実施手順

### 3-1. 内部モデルを作る

1. ProductCard を定義する
2. ProductDetail を定義する
3. CategorySummary を定義する
4. SiteContext を定義する

### 3-2. テンプレート適用層を作る

1. 共通ヘッダー / フッターの扱いを決める
2. トップページ用レンダラを作る
3. カテゴリ一覧用レンダラを作る
4. 商品詳細用レンダラを作る
5. お知らせ用レンダラを作る

### 3-3. 出力整形を作る

1. assets コピーを実装する
2. HTML 内リンクの書き換えを実装する
3. ローカル画像パスへの置換を実装する
4. not-implemented ページを作る

### 3-4. build コマンドへ統合する

1. 取得済みデータ読込
2. 正規化
3. ページ生成
4. assets 配置
5. 出力完了サマリ表示

## 4. 成果物

- output/mock-site 一式
- トップページ
- カテゴリ一覧ページ
- 商品詳細ページ
- お知らせページ

## 5. 完了判定

- 1 商品詳細ページが画像付きで開ける
- カテゴリ一覧から商品詳細へ遷移できる
- トップページが表示できる
- お知らせ一覧 / 詳細が開ける

## 6. 注意点

- MakeShop 独自構文の完全実行を目指さない
- 実ショップとの差分吸収をここで過度に追わない
