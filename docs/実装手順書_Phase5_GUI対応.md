# 実装手順書 Phase 5 GUI対応

## 1. 目的

CLI コアを再利用し、非開発者でも操作しやすい GUI を作る。

## 2. 開始条件

- Phase 4 が完了していること
- CLI build が安定していること

## 3. 実施手順

### 3-1. use case 層を整理する

1. build 実行をサービス化する
2. fetch-source をサービス化する
3. export-data をサービス化する
4. fetch-images をサービス化する
5. capture-live-site をサービス化する

### 3-2. GUI 画面を作る

1. 入力フォームを作る
2. 設定値の保存 / 再利用を作る
3. 実行ボタンを作る
4. 進捗表示を作る
5. ログ表示を作る

### 3-3. 実行連携を作る

1. GUI から use case を呼び出す
2. 実行中の状態表示を作る
3. 完了時の通知を作る
4. 出力先を開く導線を作る

### 3-4. secret 表示を制御する

1. パスワード欄を伏せ字にする
2. token を再表示しない
3. ログへ secret を出さない

## 4. 成果物

- GUI アプリ初版
- GUI 操作で build 実行可能な状態

## 5. 完了判定

- GUI から build が実行できる
- GUI と CLI の出力結果に重大差異がない
- secret 値が GUI やログに露出しない

## 6. 注意点

- GUI の都合でコアロジックを汚さない
- 画面装飾よりも実行安定性を優先する
