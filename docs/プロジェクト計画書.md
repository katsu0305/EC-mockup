# ローカルモック生成ツール プロジェクト計画書

## 1. プロジェクト概要

関連仕様:

- .env の正式仕様は env仕様書.md を参照
- Live Site Inspector の CLI 仕様は CLI仕様_LiveSiteInspector.md を参照
- フェーズ別の実装手順は 実装手順書_一覧.md を参照

### 1-1. プロジェクト名

ローカルモック生成ツール開発プロジェクト

### 1-2. 目的

GitHub 上で管理されている EC サイトテンプレートと、EC-database-tang から取得できる商品データ、ならびに MakeShop の画像資産を利用して、ローカルで閲覧可能な静的モックサイトを生成するツールを開発する。

加えて、実際のショップサイトから取得できる HTML / CSS / JS 一式を補助参照情報として使い、ローカル生成物との見た目差分確認や DOM 構造確認に利用する。

初期は CLI ツールとして構築し、後続フェーズで GUI 化および PyInstaller による単体配布を実現する。

### 1-3. 背景

- MakeShop 環境を直接触らずに、サイトの見た目と掲載商品をローカルで確認したい
- 実データに近い商品情報、価格、画像を使ってレビューできる状態を作りたい
- 会員機能や注文機能を除外した確認専用モックを短時間で再生成したい
- GitHub 上の特定時点のテンプレートを基準に、再現性のある成果物を生成したい
- 必要に応じて実ショップの見た目と生成物のズレも確認したい

### 1-4. スコープ

本プロジェクトで実装する対象:

- GitHub からのテンプレート取得
- EC-database-tang からの商品データ取得
- MakeShop 公開 URL / FTP からの必要画像取得
- 実ショップサイトの HTML / CSS / JS の補助参照
- 会員制クローズドサイトへのアクセス情報を .env から読み込む実装
- 静的 HTML モックサイト生成
- ローカルプレビュー
- CLI 実装
- GUI 実装準備
- PyInstaller による GUI 配布対応

本プロジェクトで初期対象外とするもの:

- 商品注文機能の再現
- 会員登録、ログイン、マイページの再現
- 問い合わせ送信機能
- MakeShop 独自構文の完全実行環境化
- 本番サイトと同等のバックエンド動作再現
- 実ショップ HTML の全面クロールを主生成ロジックに採用すること

### 2-3. 完了条件

以下を満たした時点を本プロジェクトの初期完了とみなす。

- CLI で build が成功する
- トップ、カテゴリ一覧、商品詳細、お知らせの主要ページが出力される
- 出力サイトがローカル静的サーバで閲覧できる
- 主要画像がローカル参照で表示される
- GUI 版がビルド処理を呼び出し、PyInstaller で実行ファイル化できる

## 5. 前提条件

- GitHub 上の対象リポジトリへアクセス可能であること
- EC-database-tang の API または DB にアクセス可能であること
- MakeShop 画像取得に必要な公開 URL または FTP 接続情報が利用可能であること
- 実ショップの会員制ログインに必要なアクセス情報を .env で保持できること
- Windows 環境での利用を前提とすること
- Python 実行環境を用いた CLI 開発が可能であること

---

## 6. 制約条件

- 初期フェーズでは MakeShop の完全再現を行わない
- 非対応機能はダミーリンクまたは未実装画面で代替する
- 画像取得は参照ベースとし、全件同期は行わない
- GitHub / API / FTP の認証情報は平文でリポジトリ管理しない
- 実ショップアクセス情報も .env でローカル管理し、リポジトリ管理しない
- GUI は CLI コアを再利用し、別実装しない

## 8. 開発方針

### 8-1. 基本方針

- CLI 先行で開発する
- ビルドロジックを GUI と分離する
- 小さく動く単位で段階的に広げる
- まず 1 商品詳細ページが正しく出る状態を最優先とする
- 実ショップ参照は補助用途に限定し、正本は GitHub テンプレートと商品データに置く

### 8-2. アーキテクチャ方針

- Source Fetcher: GitHub からテンプレート取得
- Data Exporter: 商品データ取得
- Image Fetcher: 画像取得
- Live Site Inspector: 実ショップの HTML / CSS / JS の補助参照
- Live Site Inspector は .env の認証情報を使ってクローズドサイトへアクセスする
- Normalizer: 内部モデル変換
- Static Builder: HTML 生成
- Preview Server: ローカル表示
- GUI Shell: CLI / Application Service 呼び出し

### 8-3. 品質方針

- CLI 実行結果は再現性を重視する
- GUI と CLI は同一ロジックを利用する
- 画像未取得時は明示ログを残す
- 失敗時は段階ごとに原因が分かるログを出力する

## 13. 受け入れ基準

### 機能受け入れ

- 指定した GitHub ref のテンプレートを取得できる
- 商品データを取得できる
- 必要画像を取得できる
- mock-site が output に生成される
- トップ、カテゴリ、商品詳細、お知らせをブラウザ確認できる

### 非機能受け入れ

- エラー時に原因がログで追える
- 再実行時に出力が壊れない
- GUI と CLI のビルド結果に重大差異がない

### 配布受け入れ

- PyInstaller 実行ファイルで GUI 起動できる
- 必要最低限の手順で build を実行できる

## 14. リスクと対策

### R1. GitHub 取得仕様の揺れ

リスク:

- テンプレート配置や対象パスが変わる

対策:

- subpaths を設定化する
- 取得処理をテンプレート構造に直結させすぎない

### R2. 商品データ構造の不足

リスク:

- カテゴリ情報や画像情報が十分でない

対策:

- 中間 JSON で欠損項目を可視化する
- 初期は必要最小項目に限定する

### R3. 画像取得失敗

リスク:

- URL 切れ、FTP 接続不良、容量制限

対策:

- 優先順位付き取得
- リトライ
- プレースホルダー画像
- 未取得ログ出力

### R4. テンプレート再現の複雑化

リスク:

- MakeShop 独自構文を再現しようとして工数が膨らむ

対策:

- 完全再現を目標にしない
- 最初は専用レンダラ方式に限定する

### R5. 実ショップ参照への依存過多

リスク:

- 実ショップの HTML を見すぎて、生成ロジックの正本が曖昧になる

対策:

- 実ショップ参照は差分確認と補助調査に限定する
- 正本は GitHub テンプレートと商品データに固定する

### R6. クローズドサイト認証の不安定さ

リスク:

- ログイン方式変更や cookie 期限切れで取得に失敗する

対策:

- 認証情報は .env で差し替え可能にする
- ログイン処理を設定化する
- 取得失敗時は HTML / CSS / JS ごとに原因をログへ残す

### R7. GUI 先行による停滞

リスク:

- UI とロジックの問題が混ざり、進捗が止まる

対策:

- CLI 完成後に GUI へ進む
- GUI は薄いラッパーに限定する

### R8. PyInstaller 配布時の差異

リスク:

- 開発環境では動くが配布版で動かない

対策:

- 初期から同梱ファイルの扱いを設計する
- 配布検証を終盤ではなく GUI 初版直後に一度実施する

## 18. まとめ

本プロジェクトは、EC サイトのテンプレート、商品データ、画像資産を統合して、ローカル閲覧用モックサイトを自動生成するツールを構築するものである。

成功の鍵は、以下の順で進めることにある。

- まず CLI でコア処理を成立させる
- 次に静的モック生成を安定化する
- 最後に GUI と PyInstaller 配布へ展開する

この順序を守ることで、スコープ膨張を抑えながら、実用的な成果物へ到達しやすくする。
