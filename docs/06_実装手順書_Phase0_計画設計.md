# 実装手順書 Phase 0 計画設計

## 1. 目的

実装着手前に、要件、入力元、設定仕様、作業ディレクトリ方針を固定し、後続フェーズの手戻りを減らす。

## 2. 開始条件

- 構想書が存在すること
- プロジェクト計画書が存在すること
- .env 仕様書の初版が存在すること
- Live Site Inspector の CLI 仕様書の初版が存在すること

## 3. 実施手順

### 3-1. 要件を固定する

1. 対象ページを確定する
2. 非対応機能を確定する
3. 実ショップ参照の用途を補助用途に限定する

### 3-2. 入力元を確定する

1. GitHub リポジトリと取得対象パスを確定する
2. MakeShop API の接続方法と認証方式を確定する
3. MakeShop 画像取得の優先順位を確定する
4. 実ショップ参照のログイン方式を確定する

### 3-3. 設定仕様を確定する

1. .env キー一覧を確定する
2. 設定ファイルと .env の責務分担を確定する
3. secret をログへ出さない方針を確定する

### 3-4. ディレクトリ構成を確定する

1. src 配下の責務分割を決める
2. work と output の役割を決める
3. docs 配下の運用文書を整理する

## 4. 成果物

- 構想書の更新版
- プロジェクト計画書の更新版
- .env 仕様書
- CLI 仕様書
- 初期ディレクトリ構成案

## 5. 完了判定

- 開発対象と非対象が明文化されている
- 入力元ごとの認証方法が決まっている
- .env キー一覧が確定している
- CLI 初期コマンド一覧が定義済みである

## 6. 注意点

- ここで GUI の画面詳細に入りすぎない
- 本番機能再現の話を広げない
- 実ショップ HTML を正本にしない
