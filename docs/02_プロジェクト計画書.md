## 2. プロジェクト概要

### 2-1. 目的

GitHub 上の EC サイトテンプレート、MakeShop GraphQL API、MakeShop の画像資産、実ショップの参照情報を用いて、ローカルで閲覧可能な静的モックサイトを生成する。

生成物は、会員機能や注文機能を持たない確認用サイトとして利用し、デザイン確認、商品掲載確認、レビュー、差分確認の基盤とする。

### 2-2. 現在の到達点

- CLI で `fetch-source` / `export-data` / `fetch-images` / `build` / `serve` を実行可能
- MakeShop API から公開中商品データを取得可能
- `getProductCategory` からカテゴリマスタを取得可能
- MakeShop FTP から商品画像を取得可能
- トップ、カテゴリ、商品詳細、お知らせの静的生成が可能
- 全ページ下部に購入前情報モジュールを共通表示可能
- PyInstaller による CLI 実行ファイル化を確認済み

### 2-3. 現在の主要フロー

1. GitHub からテンプレート取得
2. MakeShop API から商品・カテゴリ取得
3. MakeShop 画像取得
4. 静的 HTML 生成
5. ローカル配信とブラウザ確認

## 6. 前提条件

- GitHub 上の対象リポジトリへアクセス可能であること
- MakeShop API 認証情報を保持していること
- MakeShop 画像取得に必要な公開 URL または FTP 接続情報が利用可能であること
- 実ショップの参照に必要なアクセス情報を `.env` で保持できること
- Windows 環境での利用を前提とすること
- Python 実行環境、または PyInstaller 配布物を使用可能であること

## 7. 制約条件

- MakeShop の完全再現は行わない
- 非対応機能はダミーリンクまたは静的表示で代替する
- 画像は必要に応じて別途取得する
- GitHub / MakeShop / FTP / 実ショップの認証情報はリポジトリ管理しない
- 実ショップ参照は補助用途に限定し、生成の正本は GitHub テンプレートと MakeShop データに置く

## 8. アーキテクチャ方針

- Source Fetcher: GitHub からテンプレート取得
- Data Exporter: MakeShop API から商品・カテゴリ取得
- Image Fetcher: 商品画像取得
- Live Site Inspector: 実ショップの補助参照取得
- Normalizer: 内部モデル変換
- Static Builder: HTML 生成
- Preview Server: ローカル表示
- GUI Shell: 将来的な GUI ラッパー

## 9. 品質方針

- CLI 実行結果の再現性を優先する
- 失敗時は段階ごとに原因が分かるログを出す
- 画像未取得時は代替表示または明示ログを残す
- テンプレートの変更は `work/source` を起点に吸収する

## 14. 受け入れ基準

### 機能受け入れ

- 指定した GitHub ref のテンプレートを取得できる
- MakeShop API から公開中商品を取得できる

- `output/mock-site` が生成される

### 非機能受け入れ

- エラー時に原因がログで追える
- 再実行時に出力が壊れない
- テンプレート更新時に再ビルド可能である

### 配布受け入れ

- PyInstaller 実行ファイルで CLI 起動できる
- `.env` を同階層に置けば `export-data` が実行できる

## 15. リスクと対策

### R1. GitHub テンプレート構成変更

対策:

- `GITHUB_SUBPATHS` を設定化する
- 取得処理の依存を局所化する

### R2. MakeShop API スキーマ変更

対策:

- `searchProduct` と `getProductCategory` の依存箇所を限定する
- 中間 JSON で欠損を可視化する

### R3. 画像取得失敗

対策:

- FTP と公開 URL の二系統を維持する
- 未取得時は代替表示とログを出す

### R4. テンプレート差し込み複雑化

対策:

- モジュール単位のテンプレート読み込みを優先する
- 本番 HTML を必要以上に再実装しない

### R5. 配布版と開発版の差異

対策:

- PyInstaller 向け `.env` 探索を明確化する
- 配布物同梱対象を固定化する
