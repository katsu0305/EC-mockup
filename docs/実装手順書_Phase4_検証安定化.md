# 実装手順書 Phase 4 検証安定化

## 1. 目的

CLI ベースのモック生成を、日常利用に耐える範囲まで安定化する。

## 2. 開始条件

- Phase 3 が完了していること
- build で最低限のモックが生成できること

## 3. 実施手順

### 3-1. 異常系を洗う

1. GitHub 取得失敗を確認する
2. API 失敗を確認する
3. 画像取得失敗を確認する
4. Live Site Inspector のログイン失敗を確認する

### 3-2. ログとエラー表示を改善する

1. 段階別エラー文言を整える
2. secret マスクを再確認する
3. 失敗時の終了コードを整える

### 3-3. 性能と容量を確認する

1. build 所要時間を測定する
2. 画像取得件数を確認する
3. output 容量を確認する
4. 不要ファイルが出ていないか確認する

### 3-4. 差分確認を行う

1. 実ショップ参照との差分を確認する
2. レイアウト差異を記録する
3. 許容差異と未対応差異を分ける

### 3-5. 既知課題を整理する

1. 欠損画像
2. 不足 CSS / JS
3. 差分が大きいページ
4. GUI へ持ち越す課題

## 4. 成果物

- 安定版 CLI
- 既知課題一覧
- 差分確認メモ

## 5. 完了判定

- 一通りの build が安定実行できる
- 異常系のログで原因を追える
- 既知課題が整理されている

## 6. 注意点

- この段階で GUI へ逃げない
- 不具合の根本原因を曖昧にしない
