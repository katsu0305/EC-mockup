# 実装手順書 Phase 1 CLI基盤構築

## 1. 目的

後続機能を載せるための Python プロジェクト基盤と CLI 実行基盤を作る。

## 2. 開始条件

- Phase 0 が完了していること
- ディレクトリ構成案が確定していること

## 3. 実施手順

### 3-1. Python プロジェクトを初期化する

1. パッケージ構成を作成する
2. 依存関係管理ファイルを作成する
3. テストディレクトリを作成する

### 3-2. CLI エントリポイントを作成する

1. メイン CLI を作成する
2. サブコマンドの雛形を作成する
3. ヘルプ表示を整える

初期コマンド:

- build
- fetch-source
- export-data
- fetch-images
- capture-live-site
- serve

### 3-3. 設定読み込み基盤を作る

1. .env 読み込み処理を作る
2. 設定ファイル読み込み処理を作る
3. 必須項目バリデーションを作る

### 3-4. ログ基盤を作る

1. 標準ロガーを作る
2. ログレベル制御を入れる
3. secret マスク処理を入れる

### 3-5. 作業ディレクトリ制御を作る

1. work ディレクトリ生成処理を作る
2. output ディレクトリ生成処理を作る
3. temp ディレクトリ整理処理を作る

## 4. 成果物

- 実行可能な CLI 骨組み
- 設定ローダー
- ログ基盤
- ディレクトリ管理基盤

## 5. 完了判定

- CLI が起動する
- 各コマンドの雛形が呼び出せる
- .env と設定ファイルが読める
- ログが出る

## 6. 注意点

- 業務ロジックをこのフェーズで作り込みすぎない
- GUI 固有のコードを入れない
