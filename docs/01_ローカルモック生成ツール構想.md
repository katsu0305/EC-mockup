# ローカルモック生成ツール構想

## 1. 目的

関連仕様:

- .env の正式キー一覧は 03_env仕様書.md を参照
- Live Site Inspector の CLI 契約は 04_CLI仕様_LiveSiteInspector.md を参照
- フェーズ別の実装手順は 05_実装手順書_一覧.md を参照

本ツールは、GitHub 上の EC サイトテンプレート、MakeShop GraphQL API、MakeShop の画像資産を使って、ローカルで閲覧可能な静的モックサイトを生成するためのビルダーである。

実ショップ参照は正本ではなく、差分確認や DOM / CSS / JS 調査のための補助情報として扱う。

## 2. 再現対象と非対象

### 2-1. 再現対象

- サイトの見た目
- 掲載商品
- 価格表示
- カテゴリ導線
- 商品詳細導線
- お知らせ導線
- 購入前情報セクション

### 2-2. 再現対象外

- 商品注文
- 会員登録
- ログイン状態の完全再現
- 問い合わせ送信
- お気に入り
- レビュー投稿
- 検索機能の完全再現

これらはダミーリンク、スタブページ、静的表示で代替する。

## 3. 入力元

1. GitHub リポジトリ
   - EC サイトテンプレートソース
   - 静的アセット
   - お知らせテンプレートや補助ファイル

2. MakeShop GraphQL API
   - 公開中の商品情報
   - 商品説明、価格、在庫、カテゴリ情報
   - カテゴリマスタ

3. MakeShop 画像用 FTP / 公開 URL
   - 商品画像や運用画像の取得元

4. 実ショップサイト
   - HTML / CSS / JS の補助参照元
   - クローズドサイトのログイン後スナップショット取得対象

## 4. 基本方針

### 4-1. MakeShop を完全再現しない

MakeShop 独自テンプレート構文や会員状態、カート挙動を完全再現するのではなく、閲覧用モックとして必要なページだけを静的 HTML に変換する。

### 4-2. GitHub をテンプレートの正本にする

テンプレートは実行時に GitHub から取得して `work/source` に展開する。

### 4-3. 商品データは MakeShop API を正本にする

現行実装では商品・価格・説明・カテゴリを MakeShop GraphQL API から取得する。コード上の旧 ECDB / MSSQL 互換項目は運用上の主系ではない。

### 4-4. ビルダーは取得元を意識しない

HTML 生成処理は、取得済みテンプレートと `work/data` の JSON を入力として動く構造にする。

### 4-5. CLI をコアにし、GUI は薄く載せる

ビルド処理の本体は CLI と Python モジュールとして実装し、GUI は将来的にそれを呼び出す薄い画面層とする。

### 4-6. 画像は必要なものを優先取得する

全件同期ではなく、商品・テンプレートで必要な画像を中心に取得する。運用上は `fetch-images` を明示実行する。

## 6. 処理フロー

1. `.env` を読み込む
2. GitHub からテンプレートを取得する
3. `work/source` に展開する
4. MakeShop API から商品とカテゴリを取得する
5. 必要に応じて実ショップ参照を取得する
6. 画像を FTP または公開 URL から取得する
7. 商品データを内部モデルへ正規化する
8. テンプレートを静的 HTML に変換する
9. assets とローカル画像を配置する
10. `output/mock-site` に出力する

## 7. 商品データ取得方針

### 7-1. 取得方法

現行実装では以下を使用する。

- `searchProduct`: 商品一覧、価格、在庫、説明、カテゴリ
- `getProductCategory`: カテゴリマスタ

`products.json` には公開中商品 (`display = Y`) のみを保存する。

### 7-2. 必要データ項目

- system_code
- custom_code
- product_name
- sell_price
- consumer_price
- quantity
- pc_content
- pc_add_content
- mobile_content
- mobile_add_content
- product_group_code
- product_group_name
- categories

### 7-3. 内部モデル化

商品データは、そのままテンプレートへ流し込まず、以下の内部モデルへ変換する。

- SiteContext
- ProductCard
- ProductDetail
- CategorySummary
- NewsItem

## 8. 画像取得方針

### 8-1. 画像取得元

画像は以下の優先順で扱う。

1. GitHub 管理下の画像アセット
2. `work/images` の既取得画像
3. MakeShop の公開画像 URL
4. MakeShop の FTP 領域

### 8-2. ローカル保存方針

取得画像は `work/images` に保持し、ビルド時に `output/mock-site/work/images` へコピーする。

### 8-3. HTML 側の扱い

ローカル画像が見つかれば相対パスへ置き換え、見つからなければ公開 URL にフォールバックする。

## 13. GUI 化の方針

GUI はビルドロジックの本体ではなく、実行操作のフロントエンドとして扱う。現時点では CLI が主系であり、GUI は継続課題とする。

## 14. まとめ

本ツールは、GitHub テンプレート、MakeShop API、MakeShop 画像資産を使って、ローカル閲覧用の静的モックサイトを生成するためのビルダーである。

現在は CLI 中心の実用段階にあり、今後は GUI と配布品質の整備、運用安定化を進める。
